PHILOSOPHY

四条设计原则

  • 单文件、零构建 —— 每款一个 HTML,双击即玩,无打包无后端
  • 离线优先 —— 贴图/音效/配乐全部用代码现场生成
  • 统一美学 —— 合成波 OutRun 霓虹风,跨 34 款视觉一致
  • 可演示 —— 内建 ?demo 自动演示 与 ?record 自录
THE KEY INSIGHT
游戏类型的差异,
不在引擎,而在四个旋钮:

🎚️ 维度 · 🎚️ 是否要物理
🎚️ 输入方式 · 🎚️ 胜负结构
底座不变,换旋钮即得新类型 —— 这就是 34 款共享同一内核的根本。
FOUNDATION

  • 碰撞 圆/AABB/Box3 + 高速扫描

  • 反馈 juice 顿帧/红闪/抖动/粒子
  • 存档 localStorage / PWA / i18n

PSEUDO-3D

伪 3D 的两个招牌技巧

  • bendX(z) 弯道 —— 远处横向偏移 ∝ 曲率×距离²,直路看着就蜿蜒;近处偏移≈0,碰撞仍按车道判定
  • hillY(z) 坡道 —— 单块细分地面,每帧位移顶点形成无缝起伏;车与地共用同一函数,永远贴坡
90 年代街机赛车的经典做法,在 WebGL 上以极小代价复刻立体感。
AUDIO

音频:零文件,全合成

  • 音效/配乐 —— Web Audio 振荡器 + 包络,前瞻调度器采样级精准
  • 空间音频 —— StereoPanner 按方位播对手车"嗖"声
  • 程序化引擎声 —— 锯齿波频率随车速实时升降
  • 音画同步 —— 节奏游戏判定以 audioCtx.currentTime 对齐,非 rAF

VIDEO PIPELINE

全自动预告片管线

  • 录制 canvas → captureStream+MediaRecorder;DOM/SVG → ffmpeg gdigrab 屏幕录制
  • 整形 ffmpeg 模糊填充把竖屏游戏铺满 1080×1920
  • 画面 HyperFrames(HTML→MP4,GSAP 可 seek 时间轴逐帧渲染)
  • 旁白+配乐 系统 TTS(Huihui)+ Node 合成 WAV → ffmpeg 混音合成
核心取舍:HyperFrames 渲不了实时游戏 → 实机画面必须先录后嵌。

METHODOLOGY

多智能体编排生产

  • 建造阶段 —— N 个 agent 并行各做一款,共享风格+录制+稳健性规范
  • 审查阶段 —— 每款跟一个对抗式审查 agent 通读修硬伤(部分用 CDP 实测)
  • 共 ~50 个 agent、数百万 token;规范随每批迭代,后续几乎一次成型
把"我没法亲手跑每个游戏"的盲区,转化为可规模化的并行质检。
LIMITS

局限与权衡

  • 无真·联机 —— 单文件离线做不了真实时多人(以本地双人替代)
  • 硬件交互不可自动演示 —— 需真人在场
  • 伪 3D vs 真 3D · 自写物理 vs 引擎 —— 以简洁/性能换真实度

完整的游戏平台。
